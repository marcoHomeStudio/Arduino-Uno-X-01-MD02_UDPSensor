It needs only </span><span class="c6">one RX and One TX</span><span class="c7">&nbsp;connection to directly communicate through RS485 lines. Conventional MAX 485 uses two more lines to be able to communicate through the RS485 communication line. Rs 485 is not full duplex and this line allows you to select which mode the module is in. <br>With the Auto flow control of the MAX485, the module itself takes care of this part of the communication, simplifying the programmation. <br>However I did not try to use the Modbus RS485 protocol with the auto flow control capability and further testing must be performed to validate it.</span></p><p class="c26"><span>The silk screen printing on the module uses unconventional RX and TX labeling, therefore the </span><span class="c6">TX</span><span>&nbsp;of the module must be connected to the </span><span class="c6">TX</span><span>&nbsp;of the arduino and the </span><span class="c6">RX</span><span>&nbsp;of the module must be connected to the </span><span class="c6">RX</span><span class="c7">&nbsp;of the arduino.</span></p><p class="c26"><span>The two wires of the RS485 line must be </span><span class="c6">twisted </span><span class="c7">to work properly for a long distance.</span></p><h2 class="c24" id="h.8x8356mrozjh"><span class="c11">Remarks about the Arduino Uno UART:</span></h2><p class="c26"><span>RS485 protocol is working only on </span><span class="c6">physical UART</span><span>&nbsp;and </span><span class="c6">Software serial</span><span>&nbsp;library </span><span class="c6">can&rsquo;t be used</span><span class="c7">&nbsp;with the RS485 communication protocol.<br>The Arduino Uno got only one physical UART connected to the pin0 and 1.
